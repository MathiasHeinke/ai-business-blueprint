# Contributing

The AI Business Blueprint is open-source by design — improvements, corrections, translations, and new templates are welcome.

## Ways to contribute

### 1. Fix or improve a framework file
Found a typo, an unclear sentence, or a better way to phrase something? Open a Pull Request.

### 2. Add a translation
We currently publish in English. Translations into German, Spanish, French, Portuguese, or any other language are very welcome. Create a parallel folder, e.g. `frameworks-de/` and mirror the structure.

### 3. Add a real-world case study
If you've used the blueprint to solve a problem in your business, we'd love a short write-up to add as an appendix. Anonymized examples are fine.

### 4. Add a new prompt to the prompt library
The prompts in `prompts/ai-prompts.md` are designed to be tool-agnostic. If you've found a phrasing that works particularly well in a specific assistant, add it.

### 5. Suggest a new framework or module
The blueprint is intentionally focused. We won't add modules that duplicate existing ones, but if there's a clear gap (e.g. a fundraising module, a community-building module), open an Issue first to discuss before building.

## Workflow

1. **Fork** the repo.
2. **Create a branch** with a descriptive name (e.g. `fix/typo-in-bottleneck-finder`, `feat/spanish-translation`, `docs/case-study-saas-agency`).
3. **Make your changes.** Keep PRs focused — one improvement per PR.
4. **Open a Pull Request** with a brief description of what changed and why.
5. We aim to review within 7 days.

## Style guide

- **Plain English.** Avoid jargon, especially MBA-speak.
- **Active voice.** "You diagnose your bottleneck," not "the bottleneck is to be diagnosed."
- **Concrete examples.** Every principle should have at least one example.
- **No fluff.** No "in today's fast-paced world..." sentences. Get to the point.
- **Cite where relevant.** Quote attribution should follow the format in existing files.
- **Markdown formatting:** standard CommonMark. Headings start at `#` for the file title.

## What we won't accept

- Promotional content for products or services (own or others)
- Content that violates the spirit of the original sources
- Translations done by raw machine translation without a human pass
- "Improvements" that add complexity without adding clarity (the blueprint exists to *reduce* complexity)

## Code of Conduct

Be useful. Be kind. Disagree without being a jerk. We don't have a 10-page CoC because we shouldn't need one — but if behavior is disrupting the community, the maintainer reserves the right to remove contributions and block contributors.
