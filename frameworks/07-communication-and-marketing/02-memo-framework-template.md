# Memo Framework — The Three Building Blocks

> *"There are three pieces to this puzzle. Who do you write it to? The story so far. The recommendation."* — Sam Shvatsa

## The three required elements

### 1. Audience: who are you writing to?
**Recommendation:** Write to your coach / a person *with no context*.

Why: this forces you to include everything relevant. If you write to your team, you leave assumptions in that no one can trace.

### 2. Story so far
Everything that happened before. Chronological, factual, no judgment.

> *"Here are the seven things that happened."*

→ creates *shared understanding*. After reading, every reader knows where you stand.

### 3. Issue / Recommendation / Open Questions
- **What's the problem?**
- **What do I recommend?**
- **What questions can't I answer myself?**

## The memo structure (standard template)

```markdown
# [Memo title]

**Author:** [Name]
**Date:** [YYYY-MM-DD]
**Status:** Draft / Review / Decided
**Read time:** [X min]
**Pre-read for meeting:** [date / "no meeting"]

---

## TL;DR
[2–3 sentences. If someone reads only the start — what must they know?]

## Story so far
[Chronology of relevant events / facts. What led to this point?]

## The Issue
[Which problem / which question / which decision?]

## Options
**Option A:** [Description] — Pros, Cons
**Option B:** [Description] — Pros, Cons
**Option C:** [Description] — Pros, Cons

## Recommendation
[What I propose, and why.]

## Risks / Trade-offs accepted
[What breaks worst case? What can we live with?]

## Open questions
[What can't I decide / answer myself?]

## Next steps
[What happens after this memo? Who does what by when?]

---

## Appendix
[Data, links, tables — only if necessary]
```

## Special memo: Decision Memo

When a clear decision is needed, shorter and more decisive:

```markdown
# Decision: [Topic]

## Recommendation
[A single clear sentence.]

## Story so far
[Max 5 bullets of pre-history.]

## Reasoning
[Why this recommendation — 3 main reasons.]

## If you disagree, respond here
[Otherwise: silence = consent after 48h]

## Implementation
[Who, what, when.]
```

→ "Disagree-and-commit" variant: silence = consent.

## How to apply it

**Three workflow modes:**

| Mode | When | How |
|---|---|---|
| **Quick Memo** (10 min) | small decision | TL;DR + recommendation + story so far |
| **Standard Memo** (45 min) | medium decision | Full template |
| **Strategic Memo** (multiple hours) | major decision | Template + options analysis + appendix |

## AI workflow for fast memos

**Prompt template:**
> *"Here are my unsorted thoughts on [topic]: [...]. Structure this following our memo framework: TL;DR, Story so far, Issue, Options, Recommendation, Risks, Open questions, Next steps. Keep it tight and concrete."*

→ Output as skeleton, then revise.
